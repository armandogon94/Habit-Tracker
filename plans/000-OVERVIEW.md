## How to use this plan

1. **Each slice is a vertical, shippable increment.** Complete in order; do not start slice N+1 until slice N's "Definition of done" checklist is fully ticked.
2. **Each slice file** ([slice-00-mockup.md](slice-00-mockup.md) … [slice-11-testflight.md](slice-11-testflight.md)) follows the same 10-section template — read top to bottom before starting.
3. **Run `/build` per task**, not per slice. A slice is many tasks; one task per session keeps context clean.
4. **Skill mapping** is at the top of each slice. Invoke listed skills via `Skill` tool BEFORE writing the first line of code in that task.
5. **Do not skip RED tests.** Every task that produces code starts with a failing test (per `test-driven-development`).

**Sequential constraints (do NOT reorder):**
- 02 needs 01 (Keychain refresh round-trip needs `/refresh-mobile`)
- 03 needs 02 (real auth header)
- 05 needs 03 (online flow first, offline overlay second)
- 06, 07, 08 each need 03 or 05 (real habit data on screen)
- 09 can run anytime after 03 but is cheaper after 06 (more strings to translate at once)
- 10 can run in parallel with iOS slices 04+ (different surface, different file tree)
- 11 is last (everything must be feature-complete)

**Parallelizable opportunities** (if multiple agents are dispatched):
- Slices 04 and 05 are independent (backend vs iOS persistence). Can run in two worktrees.
- Slice 10 (web admin) is fully independent of 04–09 once 01 lands.
- Slice 09 (localization) can be a background pass in parallel with 06–08.

## Risk register (project-level)

| Risk | Impact | Likelihood | Mitigation | Owner slice |
|---|---|---|---|---|
| Liquid Glass APIs change before iOS 26 GM | High | Low | Encapsulate behind `AppTheme` protocol, swap in concrete impl | 00 |
| RRULE-aware streak rewrite breaks existing seeded data | Medium | Med | Snapshot fixtures of current behavior, parametrize tests, rollback migration ready | 04 |
| Apple TestFlight cert/signing issues block release | High | Med | Do signing + first archive in slice 11 dry-run **early** in slice 00 (before features) — see slice-00 §10 | 00, 11 |
| SwiftData migrations break offline data | Med | Med | Schema versioning + lightweight migration; test cold install + upgrade in CI sim | 05 |
| Backend mobile refresh endpoint introduces session fixation if reused incorrectly | High | Low | Token rotation on every refresh; whitelist in Redis with 7-day TTL | 01 |
| Translations diverge from source as features change | Low | High | String Catalog auto-extracts; make `xcstringstool sync` part of CI | 09 |
| Widget + app race conditions on shared App Group data | Med | Med | Single source of truth = SwiftData store in App Group container; widget is read-only | 07 |
